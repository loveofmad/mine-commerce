## 一、用户购物流程

```
┌─────────────────────────────────────────────────────────────┐
│                        用户购物流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 浏览商品                                                │
│     ├─ 首页：查看推荐商品、轮播图、分类导航                   │
│     ├─ 分类页：按分类筛选商品                                │
│     └─ 搜索：关键词搜索商品                                  │
│                                                             │
│  2. 查看商品详情                                             │
│     ├─ 查看商品图片、价格、销量                               │
│     ├─ 查看商品规格/属性                                     │
│     ├─ 查看商品详情描述                                      │
│     └─ 查看用户评价                                          │
│                                                             │
│  3. 选择规格（如有）                                         │
│     ├─ 选择SKU规格                                          │
│     └─ 调整购买数量                                          │
│                                                             │
│  4. 加入购物车 或 立即购买                                    │
│     ├─ 加入购物车：稍后统一结算                               │
│     └─ 立即购买：直接进入订单确认                             │
│                                                             │
│  5. 购物车管理                                               │
│     ├─ 查看购物车商品                                        │
│     ├─ 修改数量                                              │
│     ├─ 选择/取消选择商品                                     │
│     └─ 删除商品                                              │
│                                                             │
│  6. 订单确认                                                 │
│     ├─ 选择/新增收货地址                                     │
│     ├─ 确认商品清单                                          │
│     ├─ 选择优惠券（如有）                                    │
│     ├─ 填写备注                                              │
│     └─ 查看订单金额                                          │
│                                                             │
│  7. 提交订单                                                 │
│     └─ 生成订单，进入待支付状态                               │
│                                                             │
│  8. 支付                                                     │
│     ├─ 选择支付方式                                          │
│     └─ 完成支付                                              │
│                                                             │
│  9. 订单跟踪                                                 │
│     ├─ 待付款 → 待发货 → 待收货 → 已完成                     │
│     └─ 可取消未支付订单                                      │
│                                                             │
│  10. 评价（可选）                                            │
│      └─ 收货后可对商品进行评价                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、订单状态流转

```
┌─────────────────────────────────────────────────────────────┐
│                      订单状态流转                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  待付款 (status=0)                                          │
│     │                                                       │
│     ├─→ 已付款 (status=1)  [用户支付]                       │
│     │      │                                                │
│     │      ├─→ 已发货 (status=2)  [商家发货]                │
│     │      │      │                                         │
│     │      │      └─→ 已完成 (status=3)  [用户确认收货]     │
│     │      │                                                │
│     │      └─→ 已取消 (status=4)  [商家/用户取消]           │
│     │                                                       │
│     └─→ 已取消 (status=4)  [用户取消/超时未支付]            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 三、待开发功能需求

### 3.1 订单支付功能

#### 3.1.1 功能描述
- 用户提交订单后进入支付页面
- 支持微信支付
- 支付成功后更新订单状态

#### 3.1.2 业务逻辑
1. **提交订单**
   - 检查收货地址是否已选择
   - 检查商品库存是否充足
   - 计算订单金额（商品金额 - 优惠金额 + 运费）
   - 生成订单号（雪花算法）
   - 扣减库存
   - 清空购物车中已下单商品
   - 订单状态设为"待付款"

2. **支付流程**
   - 调用微信支付统一下单接口
   - 获取支付参数
   - 调起微信支付
   - 支付成功回调更新订单状态为"已付款"
   - 支付失败或取消不做处理，订单保持"待付款"状态

3. **超时处理**
   - 订单创建后30分钟未支付自动取消
   - 取消后恢复库存

#### 3.1.3 数据库变更
- `tb_order` 表已有所需字段，无需修改

#### 3.1.4 接口设计
- `POST /api/order/create` - 创建订单
- `POST /api/order/pay/{id}` - 发起支付
- `POST /api/order/cancel/{id}` - 取消订单
- `POST /api/order/confirm/{id}` - 确认收货

---

### 3.2 商品评价功能

#### 3.2.1 功能描述
- 用户收货后可对商品进行评价
- 支持文字评价和图片上传
- 支持评分（1-5星）
- 商品详情页展示评价列表

#### 3.2.2 业务逻辑
1. **评价条件**
   - 订单状态必须为"已完成"
   - 同一订单同一商品只能评价一次
   - 评价后不可修改

2. **评价内容**
   - 评分：1-5星
   - 文字内容：最多500字
   - 图片：最多9张

3. **评价展示**
   - 商品详情页底部显示评价列表
   - 按时间倒序排列
   - 显示评价用户昵称、评分、内容、图片

#### 3.2.3 数据库变更
- `tb_evaluate` 表已有所需字段，无需修改

#### 3.2.4 接口设计
- `POST /api/evaluate` - 提交评价
- `GET /api/evaluate/list` - 获取评价列表
- `GET /api/evaluate/check/{orderId}` - 检查是否已评价

## 四、数据验证规则

### 4.1 用户相关
| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 手机号 | 11位，1开头，第二位3-9 | 请输入正确的手机号 |
| 用户名 | 必填，2-20位 | 请输入用户名 |
| 密码 | 必填，6-20位 | 请输入密码 |

### 4.2 地址相关
| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 收货人 | 必填 | 请输入收货人姓名 |
| 手机号 | 11位手机号格式 | 请输入正确的手机号 |
| 详细地址 | 必填 | 请输入详细地址 |
| 重复检测 | 姓名+手机号+地址相同 | 该地址已存在 |

### 4.3 订单相关
| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 收货地址 | 必选 | 请选择收货地址 |
| 商品列表 | 不能为空 | 无订单商品 |
| 库存 | 库存充足 | 库存不足 |

### 4.4 评价相关
| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 评分 | 1-5星 | 请选择评分 |
| 内容 | 0-500字 | 请输入评价内容 |
| 图片 | 最多9张 | 最多上传9张图片 |
